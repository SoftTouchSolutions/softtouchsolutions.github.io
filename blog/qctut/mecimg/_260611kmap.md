## 2-Way Sync — my advice: YES, enable by default

It's already implemented via the **2-Way Sync checkbox**. When on:
- Editing expression → table refreshes immediately
- Toggling any output cell → expression field updates to canonical SOP
- Renaming a variable → expression reflects new name

The only caveat to communicate to users: the back-generated expression is **canonical SOP** (sum of minterms), not the minimized form — the minimized result always appears in the green result box.

## Debugging & Web deployment

**Desktop debugging (VS Code)**
```json
// .vscode/launch.json
{ "type": "coreclr", "request": "launch",
  "program": "${workspaceFolder}/bin/Debug/net8.0/KmapSolver.exe" }
```
Install **Ionide** extension. Use `Program.withConsoleTrace` (add to Elmish chain) to trace every `Msg` to the terminal.

**Web (WASM via Avalonia.Web)**
```bash
dotnet workload install wasm-tools
# Add a sibling KmapSolver.Browser project:
dotnet add package Avalonia.Web
# TargetFramework: net8.0-browser
# Entry: AppBuilder.Configure<App>().UseAvalonia().SetupWithSingleViewLifetime()
dotnet run --project KmapSolver.Browser   # serves on localhost
```
Then deploy the `wwwroot/` output as static files to **GitHub Pages**, **Netlify**, or **Vercel** — no server needed.
